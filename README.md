# Profit Driven Classification

## Обзор проекта
В проекте исследуется задача предсказания отклика клиентов на маркетинговую кампанию банка с акцентом на максимизацию экономической выгоды.  
Основная идея — интегрировать бизнес-логику (прибыль/затраты) непосредственно в процесс обучения и оценки модели.

---

## Постановка задачи
Объект — телефонный звонок клиенту с предложением банковского продукта.  

Необходимо предсказать:
- согласится ли клиент открыть депозит  

Ключевая особенность:
- каждый звонок имеет стоимость  
- успешная конверсия приносит прибыль  

Таким образом, задача формулируется как оптимизация прибыли, а не просто классификация.

---

## Данные
Используется датасет UCI Bank Marketing.

Признаки включают:
- социально-демографические характеристики  
- информацию о звонке  
- экономические индикаторы  

Целевая переменная:
- `y` — согласие клиента (1 / 0)

---

## ML Pipeline

Используется следующий пайплайн:

- OneHotEncoder для категориальных признаков  
- StandardScaler для числовых  
- Logistic Regression как базовая модель  

Валидация:
- Stratified K-Fold (5 фолдов)  
- учет дисбаланса классов  

---

## Бизнес-метрика

Введена функция прибыли:

- +10-20 (случайная величина) долларов за успешную конверсию (TP)  
- -1 доллар за неудачный звонок (FP)  
- 0 — в остальных случаях  

Метрика:
- средняя прибыль на одного клиента  

Это позволяет напрямую оптимизировать бизнес-результат.

---

## Оптимизация модели

### Подбор гиперпараметров
- выполнен GridSearch по коэффициенту регуляризации `C`  

### Подбор порога классификации
- реализован кастомный классификатор с настраиваемым threshold  
- найден оптимальный порог, максимизирующий прибыль  

---

## Оценка качества

Рассчитаны стандартные метрики:
- precision  
- recall  

Также введена кастомная метрика:
- PR-AUC в области recall ≥ 0.5  

Это позволяет оценивать модель в зоне, важной для бизнеса.

---

## Результаты

- модель демонстрирует стабильную прибыль на кросс-валидации  
- оптимизация порога дает больший эффект, чем настройка модели  
- recall является ключевой метрикой для данной задачи
